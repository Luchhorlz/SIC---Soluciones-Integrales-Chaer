# Modalidades de contratación

Cada servicio deberá admitir una o varias modalidades.

## Precio fijo

El prestador publica un precio conocido por una tarea determinada.

Ejemplo: instalación de un ventilador de techo.

## Presupuesto personalizado

El cliente describe el problema, agrega fotografías y recibe propuestas.

Ejemplo: remodelación completa de un baño.

## Precio por hora

Adecuado para:

* Limpieza.
* Cuidado de niños.
* Cuidado de adultos mayores.
* Asistencia personal.
* Clases.

## Precio por visita

* Diagnóstico técnico.
* Consulta profesional.

# Flujo recomendado de búsqueda

La navegación debería seguir esta estructura:

**Necesidad → categoría → subcategoría → servicio → modalidad → prestador**

Ejemplo:

**Hogar y mantenimiento → Plomería → Pérdidas de agua → Reparación a domicilio → Urgente**

También debería existir un buscador por problema donde el consumidor pueda escribir:

* “Pierde agua debajo de la cocina”.
* “La heladera no enfría”.
* “Necesito una niñera para el sábado”.
* “Quiero armar una página web”.
* “Necesito trasladar un sillón”.
* “Mi auto no arranca”.

La aplicación interpretaría la necesidad y recomendaría automáticamente la categoría y el servicio correspondiente.

# Modelo de funcionamiento del marketplace

## Concepto general

La aplicación será un marketplace donde los consumidores podrán buscar, comparar y contratar prestadores de servicios previamente registrados.

Los clientes no podrán crear publicaciones, estados, anuncios abiertos ni búsquedas laborales. Tampoco deberán publicar textos como “busco plomero”, “necesito electricista” o “quiero contratar una niñera”.

En su lugar, navegarán por las categorías disponibles o utilizarán un buscador para encontrar directamente el servicio que necesitan.
